Let's OOPs

What we want is an object {} that contains two methods "add" and "mul". So that we can do something like this:


    var sum = myCal.add(2, 3);	// 5
    var mux = myCal.mul(2, 3);	// 6

The structure of the desired object should be something like this:


    var myCal = {	
        add: function () {
            ...
        },
        mul: function () {
            ...
        }
    }

Those first set of curly brackets object is what we are really intrested in. "myCal" is just a reference to this object.

OK, we can create this object statically, but in real world we need to create this object on the fly, as the page is loaded. Check the code below:


    // self executing
    var myCal = (function() {           
        var addMethod = function(x, y) {
            return x + y;
        };
        
        var mulMethod = function(x, y) {
            var r = 0;
            
            for(var i = 0; i < y; i++) {
                r = this.add(r, x);
            }
            
            return r;
        }.bind(this);
        
        // focus here - this is the object that we want
        // code above this is just to build this object
        return {
            "add": addMethod,
            "mul": mulMethod
        }

    })();

    // once the above code gets executed
    // we get the desired object as the following
    // remember this is how our object will reside in the memory
    myCal = {   
        add: function (x, y) {
            return x + y;
        },
        mul: function (x, y) {
            var r = 0;
            for(var i = 0; i < y; i++) {
                r = this.add(r, x);
            }
            return r;
        }
    }

Check what is retured {} - an object. That's what we want. Normally novinces focus on the code above this return statement. But the return is the important part. Every code above it is only for generating this returned object {}.
